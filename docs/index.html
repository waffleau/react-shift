Motivation

Transitions in CSS are pretty easy, but not well understood. There are some gotchas too, like animating the wrong properties can lead to poor performance on mobile. On top of that, React presents an interesting challenge for transitions, as it takes this thing that's not well understood and provides an unintuitive abstraction layer on top. Declarative animations are powerful, but difficult to understand, and in many ways contrary to way you're trained to think in React.

Wax aims to create a simple and performant component library for animations, which leverages composition to make transitions easy to understand.

Installation

npm install react-wax
yarn add react-wax

Basic Usage

Components all have sensible defaults, with some having required props for when there isn\'t a sensible option available. All transitions default to 600ms. For more detailed information on options view the API reference below. 

import React from 'react'
import ReactWax from 'react-wax'

class MyComponent extends Component {
  state = {
    active: false
  }

  handleToggle = () => {
    this.setState(prevState => ({ active: !prevState.active }))
  }

  render (
    <div>
      <button onClick={this.handleToggle}>
        Toggle
      </button>
      <ReactWax.Fade active={this.state.active}>
        <YourChildComponents />
      </ReactWax.Fade>
    </div>
  )
}

API Reference

All transitions are implemented using the low-level Transition component. This component can also be used to create custom transitions if you find Wax doesn't do what you need out of the box.

The following props can be used with every transition type:

active
Type: Boolean
Default: false

Switching this prop will cause the transition to run backwards or forwards (true == forwards).

duration
Type: Number
Default: 600

How long the transition will run.

durationEnter

Same as duration but only applies when active == true.

durationLeave

Same as duration but only applies when active == false.

delay
Type: Number
Default: 0

Offset the start of the transition by the specified amount.

delayEnter

Same as delay but only applies when active == true.

delayLeave

Same as delay but only applies when active == false.

easing
Type: String or Array[4]
Default: linear
Option (Enum): linear | ease | easeIn | easeOut | easeInOut
Option (Array): Array of four numbers describing a cubic bezier - More info

Specifies the easing used by the transition.

easingEnter

Same as easing but only applies when active == true.

easingLeave

Same as easing but only applies when active == false.

style
Type: object
Default: {}

Style object that will be used on the Transition container div.
